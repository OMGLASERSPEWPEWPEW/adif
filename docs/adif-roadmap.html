Changes Roadmap

What ADIF Changes

11 areas where ADIF modernizes EverQuest's 1999 architecture — and why each change matters for players.

KEEP — still the right call UPGRADE — better options exist RETHINK — fundamentally different approach NEW — didn't exist in EQ
1

Networking

UPGRADE

EQ (1999)

  • • Custom reliability over raw UDP — hand-built sequencing, acknowledgments, fragmentation
  • • No encryption — session handshake only, no TLS, no modern auth
  • • Broadcast everything to everyone — every entity update sent to all players in the zone

ADIF (Modern)

  • • ENet or WebRTC — proven reliability libraries, no hand-rolled protocol
  • • Interest management — only send updates for nearby entities
  • • Delta compression — only send what changed since last acknowledged state
Why it matters: Supports 100+ players per zone without bandwidth explosion. EQ's "broadcast everything" approach was designed for 56k modems and 30-player raids — it doesn't scale to modern expectations.
2

Serialization

UPGRADE

EQ (1999)

  • • Raw C structs on the wire — #pragma pack(1), raw binary memcpy
  • • Opcode dispatch tables — 1-2 byte opcode maps to handler function
  • • Version-specific patch files — different struct layouts per client era, no schema evolution

ADIF (Modern)

  • • Protobuf 3 with buf linting and breaking-change detection
  • • Type-safe Packet.oneof routing — replaces brittle opcode dispatch
  • • Schema evolution — add fields without breaking old clients
Why it matters: No more silent data corruption when fields change. EQ's raw struct approach meant any change to a packet required updating both server and client in lockstep — Protobuf handles backward compatibility automatically.
3

Entity Model

RETHINK

EQ (1999)

  • • Deep inheritance hierarchy — Entity → Mob → NPC → Client
  • • Everything inherits from Mob — HP, level, race, class, position, equipment all in one massive class
  • • Type checking via downcasting — IsClient(), CastToNPC()

ADIF (Modern)

  • • Entity-Component-System (ECS) — composition over inheritance
  • • Data-oriented design — components stored contiguously in memory for cache performance
  • • Add entity types without touching existing code — just compose new component sets
Why it matters: Add new entity types without touching existing code, and get better cache performance. EQ's Mob class is thousands of lines because every entity type had to fit into the same inheritance chain.
4

Database

UPGRADE

EQ (1999)

  • • MySQL with 170+ column tables — items table has 285 columns, most NULL for any given row
  • • No normalization — flat tables with redundant data
  • • ORM-free — raw SQL queries (this part is correct)

ADIF (Modern)

  • • PostgreSQL with JSONB — variable data collapsed into queryable JSON columns
  • • Proper migrations — numbered SQL files with EQEmu comparison comments
  • • Same ORM-free approach — raw SQL for full control over query plans
Why it matters: The items table doesn't need 285 columns. JSONB collapses optional fields into a single queryable column while keeping PostgreSQL's full query engine. Migrations are versioned and documented.
5

Threading

RETHINK

EQ (1999)

  • • Single-threaded zone processes — one thread handles all game logic per zone
  • • Async I/O via libuv — non-blocking networking, but game logic is sequential
  • • Process-level parallelism only — multiple zones = multiple processes

ADIF (Modern)

  • • Rust with fearless concurrency — ownership system prevents data races at compile time
  • • async/await — efficient async I/O without callback hell
  • • No data races by construction — the compiler is the concurrency guardian
Why it matters: Modern hardware has 16+ cores. Single-threaded means 15 cores sit idle. Rust's ownership model makes multi-threaded game logic safe without the fear of race conditions that plagues C++ game servers.
6

Collision

KEEP

EQ (1999)

  • • AABB tree of triangles — .map files with axis-aligned bounding box hierarchy
  • • Line-of-sight raycasting — fast ray-triangle intersection for LoS checks
  • • Ground detection — vertical ray cast to find walkable surface

ADIF (Modern)

  • • Same approach works — AABB trees are still optimal for static geometry queries
  • • Voxel zones generate the same triangle tree — geometry format is pluggable, collision interface stays the same
  • • Proven across 25 years — battle-tested at scale
Why it matters: If it ain't broke — this design is proven across 25 years of production use. The Map interface abstracts collision from geometry format, so voxel zones slot in without changing a single line of game logic.
7

Pathfinding

KEEP

EQ (1999)

  • • Recast/Detour NavMesh — .nav files generated from zone geometry
  • • Industry standard library — same tech used by AAA engines
  • • Terrain-aware pathfinding — NPCs navigate around obstacles, up slopes, through doorways

ADIF (Modern)

  • • Same library, same approach — Recast/Detour is the gold standard
  • • Used by Unreal, Unity, and every serious game
  • • No reason to replace it — the IPathfinder interface keeps it decoupled
Why it matters: Used by Unreal, Unity, and every serious game engine. There is no better option for navmesh-based pathfinding. The IPathfinder interface means ADIF can swap implementations without affecting NPC behavior.
8

Quest Scripting

UPGRADE

EQ (1999)

  • • Embedded Perl interpreter + Lua — two scripting runtimes in the same process
  • • Event-driven model — EVENT_SAY, EVENT_DEATH, EVENT_SPAWN
  • • Perl is fragile — string manipulation bugs cause infinite loops and crashes

ADIF (Modern)

  • • Lua-only — drop Perl entirely, one fast embeddable runtime
  • • Same event model — the event-driven pattern is excellent, keep it
  • • Hot-reload + better debugging — edit quests without restarting the zone
Why it matters: Perl is fragile — a misplaced FindReplace("", "-") can cause an infinite loop. Lua is fast, embeddable, and battle-tested in game engines from World of Warcraft to Roblox. Hot-reload means content creators can iterate without waiting for zone restarts.
9

Game Loop

UPGRADE

EQ (1999)

  • • Variable tick rate (~10 Hz) — game loop runs as fast as it can, capped loosely
  • • Timer-based scheduling — events fire on wall-clock timers, not simulation ticks
  • • Non-deterministic — results vary with server load and timing

ADIF (Modern)

  • • Fixed tick rate — consistent simulation step, rendering interpolates between ticks
  • • Deterministic simulation — same inputs always produce same outputs
  • • Interpolation — smooth visuals regardless of tick rate
Why it matters: Reproducible behavior, easier debugging, fairer combat at high latency. When the game loop is deterministic, a bug report can be replayed tick-by-tick. Variable tick rates mean combat outcomes depend on server load.
10

Zone Loading

RETHINK

EQ (1999)

  • • Load entire zone at boot — all geometry, all spawn data, all scripts loaded into memory
  • • All NPCs spawned immediately — every mob in the zone exists from the moment the zone boots
  • • Fixed zone boundaries — zones are pre-defined, static containers

ADIF (Modern)

  • • Streaming/chunked loading — load geometry progressively as players explore
  • • Dynamic spawn radius — NPCs activate near players, hibernate when distant
  • • Enables larger zones — no upper bound tied to boot-time memory loading
Why it matters: Enables larger zones without 6-second boot times. EQ's approach of loading everything up front worked when zones were small — but a modern open-world zone with thousands of NPCs needs smarter resource management.
11

Inter-Process Communication

UPGRADE

EQ (1999)

  • • Custom TCP protocol — hand-rolled message passing between world and zone processes
  • • Raw struct passing — same C struct serialization used for client packets
  • • Fragile coupling — adding a new IPC message means editing multiple files

ADIF (Modern)

  • • Protobuf over TCP or gRPC — schema-defined messages with code generation
  • • Same cluster model — the zone/world/login architecture is proven, keep it
  • • Type-safe IPC — compiler catches message format errors, not runtime crashes
Why it matters: Type-safe IPC makes it easy to add new message types without the fear of breaking existing ones. The cluster model itself is sound — the wire format just needs to graduate from raw structs to a real schema.
